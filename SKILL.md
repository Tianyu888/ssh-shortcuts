# SSH Shortcuts

## Overview

Use this skill to manage SSH shortcuts as private user data. The skill provides workflow guidance and a helper script, but it must never contain real user infrastructure, passwords, private keys, internal IP inventories, production paths, or service secrets.

Default private config path:

```text
~/.config/ssh-shortcuts/shortcuts.json
```

The helper script is:

```text
scripts/ssh_shortcuts.py
```

Read `references/config-schema.md` when creating or changing the shortcut catalog format.

## Safety Rules

- Keep all real shortcut data in the user's local config, not in this skill.
- Do not store passwords, passphrases, private keys, API tokens, or root passwords in the shortcut catalog.
- Prefer SSH keys, `ssh-agent`, hardware keys, or OS keychains for authentication.
- Before committing or publishing changes, scan for private network details, credentials, and personal host inventories.
- Preserve user-managed sections of `~/.ssh/config`; only replace the managed block generated by this skill.
- Confirm with the user before running commands that connect to production systems or restart services.

## Workflow

1. Locate the private config:

   ```bash
   python3 scripts/ssh_shortcuts.py path
   ```

2. Create it if missing:

   ```bash
   python3 scripts/ssh_shortcuts.py init
   ```

3. List or inspect shortcuts:

   ```bash
   python3 scripts/ssh_shortcuts.py list
   python3 scripts/ssh_shortcuts.py show <shortcut>
   python3 scripts/ssh_shortcuts.py find-service <name-or-term>
   ```

4. Add or update a shortcut:

   ```bash
   python3 scripts/ssh_shortcuts.py add prod-api \
     --host 203.0.113.10 \
     --user ubuntu \
     --port 22 \
     --identity-file ~/.ssh/prod_api \
     --description "Production API server" \
     --service api \
     --service nginx \
     --path app=/srv/app
   ```

5. Render or install SSH config entries:

   ```bash
   python3 scripts/ssh_shortcuts.py render-ssh-config
   python3 scripts/ssh_shortcuts.py install-ssh-config
   ```

6. Test an installed shortcut:

   ```bash
   python3 scripts/ssh_shortcuts.py test prod-api
   ```

## Jump Hosts

Use the `proxy_jump` field for bastion or jump-host connections. It maps directly to `ProxyJump` in `~/.ssh/config`.

```bash
python3 scripts/ssh_shortcuts.py add internal-db \
  --host 10.0.0.12 \
  --user admin \
  --proxy-jump bastion \
  --identity-file ~/.ssh/internal_db
```

Make sure the jump host itself exists in either `~/.ssh/config` or the shortcuts catalog.
